В моем представлении должно быть так:
1. Есть состояние игры (ML, CL, MR, CR, boat_pos, MB,CB), и в начале оно имеет значение
(N,N,0,0,left,0,0).

2. Есть несколько этапов переходов по состояниям, в каждом из них надо соблюдать правила:

2.1. С левого берега - в лодку: 
(ML, CL, MR, CR, left, MB,CB) -> (ML - k, CL - l, MR, CR, left, MB + k, CB + l)
сколько взяли с берега, столько и село в лодку, причем должны соблюдаться правила:
k+l > 1 (чтобы в лодке присутствовал рулевой и пассажир), 
ML - k >= CL - l  или ML = 0 (безопасность на левом берегу),
ML - k >=0 , CL - l >= 0 (чтобы не уходить в минус),
1 <= MB + k + CB + l <= M (заполненность лодки).

2.2 С лодки - на правый берег (сразу с высадкой):
(ML, CL, MR, CR, left, MB,CB) -> (ML, CL, MR+k, CR+l, right, MB-k,CB-l)
причем высаживаться можно в любом количестве при соблюдении правил:
k+l >= 1 (чтобы точно кто-то высадился, хотя бы 1),
MR+k >= CR+l  или MR+k = 0 (чтобы было безопасно),
0 <= MR + k <= N  , 0 <= CR+l <= N , 0 <= k <= MB,  0 <= l <= CB (чтобы не уходить в минус).

2.3 С правого берега - в лодку, по правилам аналогичным 2.1:
(ML, CL, MR, CR, right, MB,CB) -> (ML, CL, MR - k, CR - l, right, MB + k, CB + l)
 k+l >= 0, k >= 0, l >= 0 (можно кого-то забрать назад, но необязательно), 
MR - k >= CR - l  или MR = 0 (безопасность на правом берегу),
MR - k >=0 , CR - l >= 0 (чтобы не уходить в минус),
1 <= MB + k + CB + l <= M (заполненность лодки).

2.4 С лодки - на левый берег (+ высадка, но необязательная), по правилам аналогичным 2.2:
(ML, CL, MR, CR, right, MB-k, CB-l) -> (ML+k, CL+l, MR, CR, left, MB-k,CB-l)
причем высаживаться можно в любом количестве при соблюдении правил:
k+l >= 0, k>=0, l>=0 (можно высаживать, но необязательно),
ML+k >= CL+l  или ML+k = 0 (чтобы было безопасно),
0 <= ML + k <= N  , 0 <= CR+l <= N , 0 <= k <= MB,  0 <= l <= CB (чтобы не уходить в минус).

3. Также надо выяснить правила для остановки перебора ситуаций (факт). По идее, наша ситуация закончится успешно, когда при высадке на правом берегу будет  ситуация 
(0,0,N,N,right,0,0). Это и будет наш факт. Он будет один, потому что других успешных завершений по условию нет, а выгрузка с лодки может быть и полной, и неполной.


4. Как объединить пункты 2.1 и 2.2 (а также по аналогии 2.3 и 2.4), чтобы исключить лодку и не плодить состояния?
(ML, CL, MR, CR, left, MB,CB) -> (ML - a, CL - b, MR + c, CR + d, right, MB + e, CB + f)
Но тут мы упираемся в постановку задачи. Считаются ли пассажиры, посаженные в лодку, находящимися на берегу? Садятся ли они все одновременно или нет? Я буду исходить из соображения о том, что пассажиры причалившей лодки видят других и могут участвовать в создании опасной ситуации. Аналогично и при посадке, пока пассажиры садятся в лодку и находятся возле берега, они причисляются к находящимся на берегу (рулевой, кстати, тоже). Исходя из такой модели, безопасная перевозка может произвестись только в том случае, когда уменьшенная группа на левом берегу безопасна и увеличенная группа на правом берегу также безопасна, то есть происходит переход
(ML, CL, MR, CR, left) -> (ML - a, CL - b, MR + a, CR + b, right)
Теперь вполне можно объединить это в один предикат с ветвлением по стороне лодки, и готово :)
